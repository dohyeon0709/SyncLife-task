## 3. AI 답을 그대로 쓰지 않고 검증·거부·수정한 사례 (중요)

- **불필요한 QueryClient 설정 발견**: Claude가 처음 mutations: { retry: false }를 명시적으로 설정하자고 제안했는데 TanStack Query 기본값이 이미 retry: 0이라 아무 동작도 안하는 죽은 코드였다. 직접 라이브러리 기본값을 확인해 new QueryClient()만 남기기로 함.
- **에러 재시도 UI의 죽은 코드 발견**: 에러 상태에서 isFetching으로 "재시도 중…" 텍스트를 보여주자는 제안을 브라우저 테스트로 검증했더니 TanStack Query가 재시도 시 status를 다시 pending으로 리셋해서 그 분기가 화면에 뜰 일이 없는 죽은 코드였고 해당 분기를 제거함.
- **경쟁 상태 처리 방식 재검토**: Claude가 처음 제안한 "요청 큐로 직렬화" 방식 대신 더 간단한 "순번 가드 + 409 시 1회 재시도" 방식으로 대체 요청. 트레이드오프 비교 후 후자로 채택. 또한 외부에서 조사해온 로컬에서 version 미리 증가시키기는 이 프로젝트의 쓰기 실패율이 15%라는 조건에서는 오히려 불필요한 충돌을 늘릴 수 있어 사용하지 않음.
- **테스트가 실제 코드를 검증하지 않는 문제 발견**: 처음 유닛 테스트를 tasks.ts에만 추가하려 했는데 Board.tsx는 그 함수들을 쓰지 않고 로직을 인라인으로 중복 구현하고 있어서 장식용 테스트가 될 뻔했다. Board.tsx가 실제로 그 함수들을 호출하도록 리팩토링한 뒤에 테스트가 실제 동작을 보장하게 됨.
- **키보드 접근성 버그 발견**: Claude가 처음 모든 카드에 tabIndex={0}을 줘서 키보드 포커스가 가능하게 만들었는데 카드가 수천 개면 다음 컬럼으로 넘어가기 위해 Tab을 카드 수만큼 눌러야 하는 실제 사용성 문제를 지적해서 잡아냄. 컬럼당 카드 하나만 Tab 순서에 두고 나머지는 방향키로 이동하는 표준 패턴으로 다시 구현함.

## 4. 스스로 판단해서 결정한 부분

- TanStack Query, react-virtual 등 라이브러리 도입 여부와 그 외 트레이드오프는 Claude가 옵션을 제시했지만 최종 선택은 직접 함.
- P2 항목 중 검색 디바운싱은 프로젝트 특성상 불필요하다고 판단해 미구현으로 결정, 대신 다중 필터를 구현.
- 시간 대비 우선순위를 고려해 다중 탭 동기화는 최종적으로 미구현으로 결정.
